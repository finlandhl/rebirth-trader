How It Works

From zero to live trading in minutes. A Binance Futures scalping bot built for speed, persistence, and survival — designed to run 24/7 on a VPS, laptop, or mobile.

01

What Is Rebirth Trader?

Rebirth Trader is a terminal-based high-frequency scalping bot for Binance Futures. It streams live prices via WebSocket across 500+ USDT + 38 USDC trading pairs, evaluates market conditions every 5 messages (EMA trend + RSI + volatility), and manages up to 20 concurrent positions — each running as its own isolated async coroutine. Every position has a 6-layer exit priority system, hedge mode support, and state persistence that survives crashes. Designed around one principle: scalp. survive. repeat.

Battle-tested with real capital across live market conditions. Every strategy, exit logic, and safeguard was developed and refined against actual Binance Futures order flow — not backtests or paper simulations.

02

Core Features

6-Layer Exit System

Stop Loss (4%), Loss Recovery Trail (3.5%+1.1%), Take Profit (6%), Trailing Stop (2.5%+0.092%), Time Limit (7d), Reversal Detection — checked in strict priority every second.

Zero-API Price Buffer

All 500+ USDT and 38 USDC pairs stream via WebSocket into in-memory deques (maxlen=120). Positions read from buffer — zero Binance API calls for monitoring. Sub-10ms latency.

Candle-Enhanced Trail

15m candle EMA(12/26) trend gate suppresses trailing stop when the trend matches the position direction — letting winners run and losers recover longer.

Hedge Mode

Dual-side positions per symbol via Binance Hedge Mode. Defensive hedge with 24h timer + candle confirmation gate prevents premature hedging.

State Persistence

Crash-safe JSONL snapshots every 20s to logs/state.jsonl. Full position restoration on restart with hedge pair linking and monitor coroutine recreation.

Systemd Integration

Auto-installs systemd service on first --persist run. Restart=always — survives SSH disconnect, reboot, and crashes.

Smart Scalping

EMA 50/100 trend detection + RSI 14 filter + volatility check. Dynamic leverage: 10x–30x based on confidence score (0.2–1.0). Entries blocked when volatility exceeds 2.5x threshold.

Symbol Blacklist

Auto-suspends symbols after 3 consecutive losses for 24h (configurable). Pre-blacklist support for known-problematic pairs. Persisted across restarts.

Trading Pause

Daily 23:00–02:00 UTC window blocks new entries and hedges during low-liquidity hours. Existing positions continue normal monitoring.

Quick Start

1. Install Dependencies

# Requires Python 3.13+ and uv (https://docs.astral.sh/uv/) cd rebirth-trader uv sync

2. Set Up API Keys

cat > .env << 'EOF' BINANCE_API_KEY=your_api_key_here BINANCE_API_SECRET=your_api_secret_here EOF chmod 600 .env

3. Console Mode (No Real Trading)

uv run streamer.py --scalp --loghfs --symbols BTCUSDC

4. Live Trading (Production)

# First run — installs systemd service + starts trading uv run streamer.py --scalp --all --live --persist --auto --ui
ℹ Auto-mode — --auto bypasses all interactive confirmations. Required for systemd / headless operation. The REBIRTH_TRADER_SYSTEMD=1 environment variable also triggers auto-mode automatically.

06

Deployment Options

🖥️ Laptop / Desktop Recommended

Run from your local machine with uv run streamer.py. Add --persist for crash recovery. The terminal connection must stay alive — use screen or tmux for persistence.

uv run streamer.py --scalp --symbols BTCUSDC --live --loghfs

Best for: testing, development, manual monitoring. No VPS needed.

☁️ VPS (Production) Best

One command auto-installs systemd: service survives reboot, SSH disconnect, and crashes. True 24/7 operation with zero manual intervention.

uv run streamer.py --scalp --all --live --persist --auto --ui --tunnel
# Health check
systemctl status rebirth-trader

Min requirements: 1 vCPU, 8GB RAM, 10GB (+50GB recommended for debug logging)

📱 Android (Termux) Experimental

Python 3.13 runs fine on Termux. Install uv via pip, sync, and run in console mode. Persistent background via termux-services or tmux.

pkg install python rust git
pip install uv
uv sync
uv run streamer.py --scalp --symbols BTCUSDC --loghfs

Best for: monitoring and testing on the go. Not recommended for full production due to Android's aggressive process management.
